# Phase 1 Plan 01: Shared Identity Database Schema Summary

**Hand-written Drizzle auth_* schema (four Better Auth tables with admin-plugin columns), an isolated drizzle/auth/ migration generated by drizzle-kit, AUTH_DATABASE_URL env wiring, and a basket_auth Postgres docker-compose service.**

## Accomplishments
- Four `pgTable` defs (`authUser`/`authSession`/`authAccount`/`authVerification`) with canonical `auth_*` names, `text` id PKs, correct camelCase->snake_case column binding, and all D-07 admin-plugin columns (`role`, `banned`, `ban_reason`, `ban_expires`, `impersonated_by`).
- Isolated `drizzle.auth.config.ts` (`out: "./drizzle/auth"`) + `db:auth:generate` script with explicit `--config` so generation never mixes with domain migrations.
- Committed SQL migration `drizzle/auth/0000_careful_iron_lad.sql` creating all four `auth_*` tables with FK cascades and unique constraints, plus `meta/` journal + snapshot.
- `AUTH_DATABASE_URL` wired through `appEnv.authDatabaseUrl` with `assertAuthDatabaseUrl()` guard, separate from Supabase env.
- `docker-compose.yml` defining the `basket_auth` Postgres 17 service with a fail-fast `AUTH_DB_PASSWORD`.
